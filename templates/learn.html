Declare variables using VAR keyword.

VAR name = "SimpLang"

ðŸ“Œ Arithmetic Operations

Supported operators: +, -, *, /, ^

VAR result = 2 + 3 * 4
VAR square = 2 ^ 4  # 2 to the power of 4

ðŸ“Œ Comparison Operators

Used in conditionals or logical checks:

==   !=   >   <   >=   <=

IF x > 5 THEN
  PRINT("Greater")
END

ðŸ“Œ Logical Operators

AND   OR   NOT

IF a > b AND b > 0 THEN
  PRINT("True")
END

ðŸ“Œ Conditional Statements

Simplang supports nested if-else:

IF x > 5 THEN
  PRINT("Big")
ELSE
  PRINT("Small")
END

ðŸ“Œ Loops

FOR loop: from start to end with optional step

FOR i = 1 TO 5 THEN
  PRINT(i)
END

FOR j = 10 TO 1 STEP -2 THEN
  PRINT(j)
END

WHILE loop: repeat while condition is true

VAR i = 0
WHILE i < 5 THEN
  PRINT(i)
  VAR i = i + 1
END

ðŸ“Œ Functions

Define and call functions with FUN keyword:

FUN square(n) -> n * n
PRINT(square(5))  # Output: 25

ðŸ“Œ Strings

VAR text = "Hello, SimpLang!"
PRINT(text)

ðŸ“Œ Lists

Lists can be created and concatenated:

VAR nums = [1, 2, 3]
VAR updated = nums + [4]
PRINT(updated)

ðŸ“Œ Constants

You can use built-in constants like:

PRINT(MATH_PI)

ðŸ“˜ Behind the Scenes: grammar.txt Insights

The grammar.txt file defines the tokens and syntax structure (BNF-style). For example:

  • VAR is defined as a keyword for variable assignment.
  • FOR and WHILE are loop tokens expecting specific structures.
  • Operators like +, - are part of arithmetic grammar rules.
  • Custom rules define how statements are parsed into an abstract syntax tree (AST).

Understanding grammar.txt helps you extend SimpLang with features like:

  • Function overloading
  • Try/Catch blocks
  • Break/Continue in loops
